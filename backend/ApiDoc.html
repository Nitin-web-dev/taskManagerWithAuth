the folder structure

the folder structure is simple and follow mvc architecture(model view controller)

  • config

    • this file contain detail and function for connection to mongodb by mongoose(odm)

  • controller

    • this file contain detail and function to controll flow and bussiness logic and this file is responsible for direct connection with database(CRUD operation)

  • helper

    • this file has some helping function such as joi a validation third party module to check req.body data
    this is used as a middleware if this function run successfully then next controller will call if any error occur
    then it will return from here

  • model

    • this file contain structure part of database (schema (blueprint))

  • node_modules

    • this is modules folder all modules file will be in it

  • routes

    • this is routes file all api's routes will be in this and on which routes which validation, function and controller should call

  • .env

    • environment file containe all secret key or variable and value
    *note this file will not be in any github repo
    you have to manually create it in root folder like (.env) and it it
    store this data
    MONGO_URL=your api without any quote or space in it
    PORT=port of your app (on which port your app is running)

  • .gitignore

    • this file will tell git to ignore those files which are listed in it

  • apidoc.html

    • this file is a doc for api reference and detail for all app working

  • app.js

    • this is main file the root or starting point of app

  • package-lock.json && package.json

    • this file contain all package list and dependency and other meta data detail about app
